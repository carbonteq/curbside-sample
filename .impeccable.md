## Design Context

### Users
Healthcare professionals (physicians, nurses, clinical staff) at hospitals and health systems. They use Curbside Health daily — often multiple times per day — to find, create, and manage clinical decision-support content: care pathways, protocols, guidelines, policies, and forms. The context is time-pressured: clinicians open the Library between patient encounters to look something up or create a new protocol. First-timers include newly onboarded clinical staff who land on the Library with no orientation.

### Brand Personality
Trusted clinical partner. Three words: **calm, precise, reliable**. The interface should feel like a well-organized reference tool that a senior clinician designed for themselves — not like generic SaaS. It should inspire confidence, not wow.

### Aesthetic Direction
Light mode. The clinical setting is bright offices and patient rooms. Cool-tinted neutrals (the existing grey scale has a subtle purple tint — good). Primary purple (#6846E6) as a single accent, used sparingly. Montserrat display / Lato body — these are the existing brand fonts and changing them is a separate initiative. Anti-reference: anything that looks like a productivity-SaaS dashboard (Notion, Linear). Also anti-reference: consumer healthcare apps (too soft, too rounded, too blue).

### Design Principles
1. **Speed first** — every interaction should feel instant. Clinicians operating at pace don't tolerate input lag or visual weight.
2. **Progressive disclosure** — show the simple path clearly; reveal complexity only when needed.
3. **Zero cognitive overhead on navigation** — one search, one primary action, clear hierarchy in every view.
4. **Earn every pixel** — no decorative elements that don't carry information. No AI banners if they're not contextually relevant.
5. **Trust through consistency** — every state (empty, filtered, selected, error) should be designed, not defaulted.
